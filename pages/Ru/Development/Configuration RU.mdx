# Документация по конфигурации Asya

В Asya есть два способа настройки параметров: **переменные среды** и **конфигурационный файл** `config.lua`, конфигурационный файл необходимо поместить по пути `$HOME/.config/asya/config.lua`. Все параметры имеют значения по умолчанию, которые описаны в конфигурационном файле, но эти значения также жестко прописаны в коде конфигурации. Поэтому, если вы пропустите какой-либо параметр, Asya выдаст предупреждение, но продолжит работать с параметрами по умолчанию.

На данный момент в Asya доступны следующие параметры конфигурации:

## Параметры конфигурации

| Table         | Field              | Переменная среды     | Значение по умолчанию     | Описание |
|---------------|--------------------|----------------------|---------------------------|----------|
| **net**       | `ws_port`        | `NET_WS_PORT`      | 3001                      | Порт, на котором работает WebSocket сервер. |
| **net**       | `ws_ip`        | `NET_WS_IP`      | "127.0.0.1"                     | IP, на котором работает WebSocket сервер. |
| **logging**   | `place`            | `LOGGING_PLACE`      | false                     | Флаг, указывающий, нужно ли выводить имя файла, где был вызван метод логгирования. |
| **logging**   | `folder`           | `LOGGING_FOLDER`     | "logs"                    | Папка, в которой будут храниться логи. |
| **logging**   | `filescount`       | `LOGGING_FILESCOUNT` | 5                         | Количество лог-файлов, которые будет хранить система. |
| **logging**   | `level`            | `LOGGING_LEVEL`      | "info"                    | Уровень логирования (например, "debug", "info", "warn", "error"). |
| **logging**   | `stdout`           | `LOGGING_STDOUT`     | true                      | Флаг, указывающий, нужно ли выводить логи в стандартный вывод (консоль). |
| **ai**        | `prompts_path`     | `AI_PROMPTS_PATH`    | "./ai-prompts.yaml"       | Путь к файлу с промптами для AI. |
| **ai**        | `groq_token`       | `AI_GROQ_TOKEN`      | "NOT"                     | Токен для доступа к API Groq (система распознавания команд и генерации ответов). |
| **plugins**   | `plugins_folder`   | `PLUGINS_FOLDER`     | "plugins"                 | Папка, содержащая плагины. |
| **plugins**   | `config`           | `PLUGINS_CONFIG`     | -                         | Конфигурация плагинов, представленных в виде словаря. |
| **open_apps** | `terminal`         | `OPEN_APPS_TERMINAL` | -                         | Путь к терминалу, который нужно открыть. |
| **open_apps** | `browser`          | `OPEN_APPS_BROWSER`  | -                         | Путь к браузеру, который нужно открыть. |

## Примечания к параметрам

1. **net**:
   - Параметры сетевой конфигурации отвечают за сетевые настройки сервера.
     - `ws_port` - порт, на котором доступен WebSocket сервер, принимающий запросы. По умолчанию установлен в `3001`.
     - `ws_ip` - ip адрес, к которому будет привязан WebSocket. По умолчанию установлен на `localhost` (`127.0.0.1`).
     {/* <!-- - `proxy_addr` - адрес прокси (будет реализовано в будущем) --> */}

2. **logging**:
   - Конфигурация логирования включает параметры, определяющие, куда и как будет выводиться информация о работе системы:
     - `place`: Указывает, нужно ли выводить имя файла, где был вызван метод логгирования. Например: ```src/logging.rs:24: 2025-01-05 00:13:21 SERVER INFO >>> Logging level: DEBUG```
     - `folder`: Папка, в которой будут храниться файлы логов.
     - `filescount`: Количество лог-файлов, которое будет храниться в папке. (будет реализовано в будущем)
     - `level`: Уровень логирования. Например, `info` для стандартных сообщений, `debug` для отладки и т. д.
     - `stdout`: Указывает, нужно ли выводить логи на стандартный вывод (в консоль).

3. **ai**:
   - В разделе конфигурации AI задаются параметры для работы с системой искусственного интеллекта:
     - `prompts_path`: Путь к файлу с предустановленными промптами для взаимодействия с AI.
     - `groq_token`: Токен для доступа к сервису Groq.


     {/* <!-- - `recognize_method`: Метод распознавания пользовательского ввода, который может быть настроен на использование `Groq`, `AltaS` или отключение AI (`None`). -->
     <!-- - `alta_s_addr`: Адрес, на котором будет размещена модель AltaS. -->
     <!-- - `autolaunch_alta_s`: Флаг для автоматического запуска модели AltaS. -->
     <!-- - `alta_s_path`: Путь к файлу модели AltaS, если она должна быть запущена автоматически. --> */}


4. **plugins**:
   - Параметры плагинов позволяют настроить использование плагинов в системе:
     - `plugins_folder`: Папка, где находятся плагины.
     - `config`: Словарь с конфигурациями для каждого плагина.

5. **open_apps**:
   - Настройки для открытия приложений при запуске:
     - `terminal`: Путь к терминалу.
     - `browser`: Путь к браузеру.

## Пример конфигурации

```lua
config = {
    net = {
        ws_port = 8080,
        ws_ip = "0.0.0.0"
    },
    logging = {
        place = true,
        folder = "/var/logs",
        filescount = 10,
        level = "debug",
        stdout = true
    },
    ai = {
        prompts_path = "./ai/prompts.yaml",
        groq_token = "your_token_here"
    },
    plugins = {
        plugins_folder = "./custom_plugins",
        config = {
            plugin1 = {
                option1 = "value1",
                option2 = "value2"
            }
        }
    },
    open_apps = {
        terminal = "/usr/bin/gnome-terminal",
        browser = "/usr/bin/firefox"
    }
}
```
