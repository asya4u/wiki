## ğŸš€ Technical Details

The project features a modular structure and consists of two main components:

1. **Daemon**  
   This is a background process responsible for all of Asya's internal logic, such as opening applications, controlling music, and more. The daemon is written in **Rust** and runs at system startup.

2. **Client**  
   This is Asya's user interface, through which you can interact with the assistant. The client communicates with the daemon via WebSocket. Currently, the client is under development. If you'd like to help, feel free to use any tech stack you prefer!
